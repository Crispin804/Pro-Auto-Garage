# Pro Auto Care

A professional web presence for Pro Auto Care, a premium automotive specialist center specializing in mechanical overhauls, custom bodyworks, 24/7 roadside recovery, and structured insurance claims processing.

---

## Project Overview

This project is a multi-page static website designed to showcase workshop services, highlight structural restoration proof via a visual portfolio, provide immediate emergency dispatch channels, and facilitate corporate fleet or insurance booking inquiries.

---

## Project Structure

* **index.html**: The landing page featuring the primary brand positioning, localized garage value propositions, and a service capability summary.
* **about.html**: Detailed information regarding the workshop's background, regional operational scale, core values, and team mechanical expertise.
* **services.html**: A complete technical directory detailing specialist diagnostic systems, mechanical bay operations, and fleet support packages.
* **gallery.html**: A rich visual portfolio utilizing live imagery with interactive hover panels alongside a custom before-and-after accident damage restoration grid.
* **contact.html**: Location mapping for the Nairobi Industrial Area workshop, operating hours, interactive map integration, and a direct inquiry form.

## Getting Started

Clone this repository on your local machine:

```bash
git clone https://github.com/Crispin804/Automotive-solutions.git
cd Automotive-solutions

```

Open index.html in any modern browser to view the site.
No build tools or dependencies are required for this static version.

## Tech Stack

* **HTML5**: Semantic structure for regional search engine optimization (SEO) and web accessibility.
* **Tailwind CSS**: Utility-first styling framework compiled via CDN for fluid, responsive layout designs.
* **Font Awesome 6.5.1**: High-definition vector icons mapping standard technical alignments across all user action prompts.
* **Markdown**: For clear project documentation.

## How to Collaborate

We welcome contributions! To maintain code quality and project integrity, please follow these steps:

### The Workflow

1. **Fork the Repository**: Create your own copy of the project to work on.
2. **Create a Feature Branch**:
```bash

```



git checkout -b feature/YourFeatureName

```
3.  **Commit Your Changes**: Use descriptive commit messages (e.g., `feat: add social media icons to footer`).
4.  **Push to Branch**:
    ```bash
git push origin feature/YourFeatureName

```

5. **Open a Pull Request (PR)**: Describe your changes clearly and link any related issues.

### Contribution Guidelines

* **Maintain Semantics**: Keep the HTML structure semantic. Avoid generic container blocks where tags like main, section, nav, or header are structurally appropriate.
* **Code Style**: Ensure all design utility classes align with the premium brand guidelines (deep navy and safety orange accents) and that tags are properly closed.
* **Documentation**: If you add a new sub-page or a significant feature, update the "Project Structure" section of this README file.

### Reporting Issues

If you find a bug or have a suggestion:

* Check the Issues tab to see if it has already been reported.
* If not, open a new issue with a clear title and a detailed description of the problem or feature request.
